Promises


Promises are a Software Design Pattern.

A promise represents the result of an operation.
This operation may be long lived, and is not guaranteed to be completed.

Rather than synchronously blocking for the long lived operation, the called function returns an object representing the promised result.

Consider the example of a third party API call across HTTP. Latency is unpredictable - HTTP is many orders of magnitude slower than cache.

When using promises, the application is free to do other units of work until the promised value is needed.

(Asynchronous Programming 101) 

            var resultsPromise = data_service.do_work(data).then(filterResults);
            emailResults(resultsPromise);

We are able to return a value ( promise )
